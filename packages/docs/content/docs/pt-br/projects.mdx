# Projetos

Um projeto é uma pasta no seu computador que o OpenChamber acompanha — geralmente uma base de código. Trocar de projeto troca a pasta em que o agente trabalha, junto com as sessões e configurações daquele projeto.

## Adicionar um projeto

Você pode adicionar um projeto a partir de alguns lugares:

- a entrada **Add project** na paleta de comandos
- o botão **+** no topo da barra lateral de sessões
- o navegador de pastas quando você escolhe um diretório

Aponte-o para uma pasta e o OpenChamber a memoriza. O nome vem da pasta; você pode mudá-lo depois.

## Trocar de projeto

Escolha um projeto na barra lateral para torná-lo ativo. Tudo — sessões, git, notas — segue o projeto que você tem aberto.

## Tornar um projeto reconhecível

Abra **Settings → Projects** para dar a um projeto um nome, cor ou ícone personalizado. O OpenChamber tenta encontrar um ícone automaticamente — veja [Ícones de projeto](/pt-br/project-icons/).

> No VS Code, o OpenChamber sempre usa a pasta que você tem aberta como o único projeto, então não há nada para adicionar ou trocar. A página de configurações de Projects fica oculta lá.

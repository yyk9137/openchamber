# Conexão com o OpenCode

Se o OpenChamber carrega mas nunca passa de "OpenCode is restarting", ou o chat não responde, o servidor com que ele conversa não está acessível. Trabalhe por estes passos.

## Está travado em "OpenCode is restarting"

- dê um momento logo após a inicialização — esse estado é normal enquanto o servidor inicia
- verifique se o servidor está vivo com `openchamber status`
- reinicie-o com `openchamber restart`
- veja os detalhes de inicialização com `openchamber logs`

## Você está conectando ao seu próprio servidor

Se você configurou o OpenChamber para usar um servidor existente, confira novamente a configuração de [OpenCode Server](/pt-br/opencode-server/):

- `OPENCODE_HOST` deve incluir a porta, sem caminho — por exemplo `http://localhost:4096`
- defina `OPENCODE_SKIP_START=true` para que o OpenChamber não inicie também o seu próprio
- se o endereço for inválido, o OpenChamber o ignora e inicia o seu próprio servidor em vez disso — procure por um aviso `[config]` nos logs

## Ainda falhando

- confirme que o Node.js é versão `20` ou mais recente
- reinstale a CLI mais recente
- abra `http://localhost:3000` diretamente antes de testar qualquer túnel ou link remoto

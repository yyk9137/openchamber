# Fluxos de Git e GitHub

O OpenChamber tem uma visão de git integrada para você revisar mudanças, fazer commit e gerenciar branches sem mudar para um terminal. Abra-a pela aba **Git** na barra lateral direita.

## Revisar e fazer commit

A visão de git divide suas mudanças em **staged** e **unstaged**:

- clique no **+** de um arquivo para fazer stage dele, ou **−** para remover do stage
- faça stage ou unstage de tudo de um grupo de uma vez
- clique em um arquivo para ver seu diff

Então escreva uma mensagem de commit e faça o commit. Você pode fazer o OpenChamber **gerar uma mensagem de commit** a partir das suas mudanças em stage — ele usa o modelo da sessão atual, então você precisa de uma sessão aberta.

## Branches e histórico

A visão de git também cobre o resto do dia a dia do git:

- criar, trocar, renomear e excluir branches
- push, pull e fetch
- navegar pelo histórico e diffs por commit
- guardar (stash) e restaurar mudanças

## Pull requests

Conecte o GitHub (veja [Issues e PRs do GitHub](/pt-br/github/)) e a aba **PR** permite abrir um pull request, atualizá-lo, marcá-lo como pronto ou fazer merge — e gerar seu título e descrição da mesma forma que as mensagens de commit.

## Trazendo conflitos

Se um merge, rebase ou integração encontra um conflito, o OpenChamber mostra o que está travado e permite resolvê-lo — incluindo entregá-lo ao agente.

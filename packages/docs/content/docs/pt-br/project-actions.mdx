# Ações de Projeto

Uma ação de projeto é um comando de shell que você salva uma vez e executa com um clique — seu servidor de desenvolvimento, um build, uma rodada de testes. Cada projeto mantém sua própria lista. Configure-as em **Settings → Projects → Project Actions**.

## Adicionar uma ação

1. Abra **Settings → Projects** e encontre a seção **Project Actions**.
2. Adicione uma ação, dê a ela um nome, escolha um ícone e insira o comando a ser executado.
3. Salve.

Você pode limitar uma ação a determinados sistemas operacionais se um comando só faz sentido em um deles.

## Executar uma ação

As ações ficam em um menu no cabeçalho do app. Clique em uma e o OpenChamber a executa em um terminal na pasta do seu projeto, alternando você para a visão de terminal para que possa acompanhar a saída. Pare-a pelo mesmo menu.

## Abrir um servidor de desenvolvimento automaticamente

Ative **auto-open URL** para uma ação que inicia um servidor. O OpenChamber observa a saída em busca de um endereço local e oferece abri-lo — veja [Preview e Servidores de Desenvolvimento](/pt-br/preview/). No desktop, você também pode roteá-lo por um encaminhamento de porta SSH.

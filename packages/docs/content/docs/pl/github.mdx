# Issues i PR na GitHub

Połącz swoje konto GitHub, a OpenChamber może pobierać issues i pull requesty, rozpocząć sesję bezpośrednio z jednego z nich oraz otwierać lub aktualizować za Ciebie PR-y.

## Połącz GitHub

1. Otwórz **Settings → Git**.
2. W sekcji GitHub wybierz **Connect**. OpenChamber pokazuje link i krótki kod.
3. Otwórz link, wpisz kod i zatwierdź.

Po połączeniu Twoje konto pojawia się w sekcji GitHub. Możesz połączyć więcej niż jedno konto i przełączać się między nimi albo rozłączyć w dowolnym momencie.

## Rozpocznij pracę od issue lub PR

Gdy tworzysz [sesję worktree](/pl/worktrees/) z połączonym GitHub, możesz wybrać **Start from GitHub issue/PR**:

- wybierz **issue**, a OpenChamber nazwie gałąź jego nazwą i otworzy sesję z issue oraz jego komentarzami jako pierwszą wiadomością
- wybierz **pull request**, a wyewidencjonuje on gałąź PR; możesz dołączyć diff PR, aby agent miał całą zmianę

To wprowadza Cię prosto do sesji z już wczytanym kontekstem.

## Otwieraj pull requesty i nimi zarządzaj

Z zakładki **PR** w [widoku git](/pl/git/) możesz utworzyć pull request, zaktualizować go, oznaczyć szkic jako gotowy lub scalić go. OpenChamber może wygenerować tytuł i opis PR na podstawie Twoich zmian.

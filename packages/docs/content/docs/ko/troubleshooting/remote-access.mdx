# 원격 접속

휴대폰이나 다른 컴퓨터에서 OpenChamber에 접근할 수 없을 때, 해결 방법은 연결 방식에 따라 다릅니다.

## 먼저 기본 사항 확인

- 같은 컴퓨터에서 `http://localhost:3000`을 먼저 엽니다. 이것이 실패하면 원격 문제가 아닙니다. [OpenCode 연결](/ko/troubleshooting/opencode-connection/)을 참고하세요
- `openchamber status`로 서버가 실행 중인지 확인합니다

## 터널 링크가 작동하지 않음

- `openchamber tunnel status --all`을 실행합니다
- 같은 인스턴스와 포트에서 터널을 재시작합니다
- 이전 링크가 이미 사용되었다면 연결 링크를 다시 생성합니다

전체 설정은 [Tunnels](/ko/tunnels/)를 참고하세요.

## 원격 인스턴스가 연결되지 않음(데스크톱)

[원격 인스턴스](/ko/remote-instances/)가 멈추면 OpenChamber가 실패한 단계를 알려줍니다.

- **auth** — SSH 또는 UI 비밀번호가 거부됨. 다시 입력하세요
- **install / start** — OpenChamber가 원격 컴퓨터에서 서버를 설정하거나 시작하지 못함. 그 컴퓨터의 요구 사항을 확인하세요
- **forwarding** — 연결은 되었지만 포트가 도달하지 않음. 다른 로컬 포트를 시도하세요

## 자체 서버 뒤에 있음

OpenChamber를 리버스 프록시 뒤에 두었는데 이상하게 로드되거나 연결되지 않으면 [Reverse Proxy](/ko/reverse-proxy/)를 참고하세요.

# Git 아이덴티티

git 신원은 커밋에 서명되는 이름과 이메일입니다. 개인 저장소와 업무 저장소를 오가며 작업한다면, 하나의 전역 설정에 의존하는 대신 신원을 저장해 두고 저장소마다 알맞은 것을 적용할 수 있습니다. **Settings → Git**에서 관리합니다.

## 신원 추가하기

1. **Settings → Git**를 열고 **New**를 선택합니다.
2. 커밋에 사용할 **name**과 **email**을 입력합니다.
3. 원격과 인증하는 방식을 선택합니다.
   - **SSH** — SSH 키를 지정합니다
   - **token** — 호스트에 저장된 자격 증명을 사용합니다
4. 필요하다면 알아보기 쉽도록 색상과 아이콘을 지정합니다.

시스템의 전역 신원도 읽기 전용으로 표시됩니다.

## 저장소에 신원 적용하기

신원을 적용하면 해당 저장소의 **local** git 설정에 기록됩니다. 전역 설정이 아니라 그 저장소에만 영향을 줍니다. SSH 신원은 사용자의 키를 사용하도록 SSH 명령도 설정하고, token 신원은 호스트용 자격 증명 저장소를 설정합니다.

OpenChamber가 기존 git 자격 증명에서 찾아낸 신원을 가져와 token 신원으로 저장할 수 있습니다.

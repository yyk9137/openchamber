# Git 및 GitHub 워크플로

OpenChamber에는 내장 git 보기가 있어 터미널로 전환하지 않고도 변경 사항을 검토하고, 커밋하고, 브랜치를 관리할 수 있습니다. 오른쪽 사이드바의 **Git** 탭에서 엽니다.

## 검토 및 커밋

git 보기는 변경 사항을 **staged**와 **unstaged**로 나눕니다.

- 파일의 **+**를 클릭해 스테이징하거나 **−**를 클릭해 스테이징을 해제합니다
- 그룹의 모든 항목을 한 번에 스테이징하거나 해제합니다
- 파일을 클릭해 diff를 봅니다

그런 다음 커밋 메시지를 작성하고 커밋합니다. OpenChamber가 스테이징된 변경 사항에서 **커밋 메시지를 생성**하게 할 수 있습니다. 현재 세션의 모델을 사용하므로 세션이 열려 있어야 합니다.

## 브랜치와 히스토리

git 보기는 일상적인 나머지 git 작업도 다룹니다.

- 브랜치 생성, 전환, 이름 변경, 삭제
- push, pull, fetch
- 히스토리와 커밋별 diff 탐색
- 변경 사항 stash 및 복원

## 풀 리퀘스트

GitHub를 연결하면([GitHub Issues & PRs](/ko/github/) 참고) **PR** 탭에서 풀 리퀘스트를 열고, 업데이트하고, 준비 완료로 표시하거나 병합할 수 있습니다. 그리고 커밋 메시지와 같은 방식으로 제목과 설명을 생성할 수 있습니다.

## 충돌 가져오기

병합, 리베이스, 통합이 충돌하면 OpenChamber가 무엇이 막혔는지 보여주고 해결하게 해줍니다. 에이전트에게 넘기는 것도 포함됩니다.

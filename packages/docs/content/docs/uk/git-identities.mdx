# Git-ідентичності

Git-ідентичність — це ім'я та email, якими підписані ваші коміти. Якщо ви працюєте і з особистими, і з робочими репозиторіями, ви можете зберегти ідентичності й застосовувати правильну для кожного репозиторію, замість покладатися на єдине глобальне налаштування. Керуйте ними в **Settings → Git**.

## Додавання ідентичності

1. Відкрийте **Settings → Git** і виберіть **New**.
2. Уведіть **name** та **email**, із якими комітити.
3. Виберіть, як вона автентифікується з віддаленим репозиторієм:
   - **SSH** — спрямуйте її на SSH-ключ
   - **token** — використовуйте збережені облікові дані для хоста
4. За бажанням дайте їй колір та іконку, щоб її було легко впізнати.

Глобальна ідентичність вашої системи теж показується, лише для читання.

## Застосування ідентичності до репозиторію

Застосування ідентичності записує її в **локальний** git-конфіг цього репозиторію — це впливає лише на цей репозиторій, не на ваше глобальне налаштування. SSH-ідентичності також задають SSH-команду для використання вашого ключа; token-ідентичності налаштовують збереження облікових даних для хоста.

Ви можете імпортувати ідентичності, які OpenChamber виявляє з ваших наявних git-облікових даних, і зберегти їх як token-ідентичності.

# Дії проєкту

Дія проєкту — це shell-команда, яку ви зберігаєте один раз і запускаєте кліком: ваш dev-сервер, збірка, прогін тестів. Кожен проєкт має власний список. Налаштуйте їх у **Settings → Projects → Project Actions**.

## Додавання дії

1. Відкрийте **Settings → Projects** і знайдіть розділ **Project Actions**.
2. Додайте дію, дайте їй назву, виберіть іконку й уведіть команду для запуску.
3. Збережіть.

Ви можете обмежити дію певними операційними системами, якщо команда має сенс лише на одній.

## Запуск дії

Дії живуть у меню в заголовку застосунку. Клікніть одну, і OpenChamber запустить її в терміналі у вашій теці проєкту, перемкнувши вас на вигляд терміналу, щоб ви могли спостерігати за виводом. Зупиніть її з того самого меню.

## Автовідкриття dev-сервера

Увімкніть **auto-open URL** для дії, що запускає сервер. OpenChamber слідкує за виводом на предмет локальної адреси й пропонує відкрити її — див. [Перегляд і dev-сервери](/uk/preview/). На десктопі ви також можете спрямувати її через SSH-перенаправлення порту.

# Віддалений доступ

Коли ви не можете дістатися OpenChamber із телефона чи іншої машини, виправлення залежить від того, як ви підключаєтеся.

## Спершу перевірте базове

- відкрийте `http://localhost:3000` спочатку на тому самому комп'ютері — якщо це не вдається, то проблема не у віддаленому доступі; див. [Підключення до OpenCode](/uk/troubleshooting/opencode-connection/)
- переконайтеся, що сервер запущений, командою `openchamber status`

## Посилання тунелю не працює

- виконайте `openchamber tunnel status --all`
- перезапустіть тунель з того самого інстансу й порту
- перегенеруйте посилання для підключення, якщо попереднє вже було використане

Повне налаштування див. у [Тунелях](/uk/tunnels/).

## Віддалений інстанс не підключається (десктоп)

Коли [віддалений інстанс](/uk/remote-instances/) застрягає, OpenChamber називає крок, який збоїв:

- **auth** — ваш SSH- чи UI-пароль було відхилено; уведіть його повторно
- **install / start** — OpenChamber не зміг налаштувати чи запустити сервер на віддаленій машині; перевірте вимоги тієї машини
- **forwarding** — з'єднання встановлено, але порт не доходить до вас; спробуйте інший локальний порт

## За власним сервером

Якщо ви поставили OpenChamber за зворотним проксі, і він завантажується дивно чи не підключається, див. [Зворотний проксі](/uk/reverse-proxy/).

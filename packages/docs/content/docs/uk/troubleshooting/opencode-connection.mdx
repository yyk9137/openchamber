# Підключення до OpenCode

Якщо OpenChamber завантажується, але ніяк не виходить за межі стану «OpenCode перезапускається», або чат не відповідає, сервер, із яким він спілкується, недоступний. Пройдіться по цьому.

## Він застряг на «OpenCode перезапускається»

- дайте йому хвилину одразу після запуску — цей стан нормальний, поки сервер стартує
- перевірте, чи сервер живий, командою `openchamber status`
- перезапустіть його командою `openchamber restart`
- перегляньте деталі запуску командою `openchamber logs`

## Ви підключаєтеся до власного сервера

Якщо ви налаштували OpenChamber на використання наявного сервера, ще раз перевірте налаштування з [OpenCode Server](/uk/opencode-server/):

- `OPENCODE_HOST` має містити порт, без шляху — наприклад, `http://localhost:4096`
- задайте `OPENCODE_SKIP_START=true`, щоб OpenChamber не запускав ще й свій
- якщо адреса недійсна, OpenChamber ігнорує її й натомість запускає власний сервер — шукайте попередження `[config]` у журналах

## Усе ще не працює

- переконайтеся, що версія Node.js — `20` або новіша
- перевстановіть найновіший CLI
- відкрийте `http://localhost:3000` напряму, перш ніж тестувати будь-який тунель чи віддалене посилання

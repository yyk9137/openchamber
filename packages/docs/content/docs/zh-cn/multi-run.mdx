# Multi-run

Multi-run 从一个表单启动多个会话 — 适合用不同的模型尝试同一个任务并比较结果。从会话侧边栏顶部的按钮打开它。

## 启动一次 multi-run

1. 打开 multi-run 启动器。
2. 选择项目并为运行组命名。
3. 编写提示词并选择要用来运行它的模型（每组最多五个）。
4. 选择是否 **isolate runs**。
5. 启动。

每个模型都会获得自己的会话，它们都以你的提示词开始。

## 隔离运行

开启 **isolate runs** 可以为每次运行提供它自己的 [worktree](/zh-cn/worktrees/) 和分支，这样它们就绝不会触碰相同的文件。这需要一个 git 仓库 — 对于不是 git 仓库的文件夹，它会自动关闭。选择运行从哪个分支开始。

关闭隔离后，每次运行都是项目文件夹中的一个普通会话。

## 比较结果

每次运行都是一个普通会话，你可以打开、阅读，并保留或丢弃它。如果你启动多个运行是为了比较不同方案，可以并排审阅它们，并把最好的那个推进下去。

如果单次运行启动失败，其余的仍会启动 — 你只会看到比请求数量更少的会话。

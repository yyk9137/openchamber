# Proyectos

Un proyecto es una carpeta de tu computadora que OpenChamber rastrea, normalmente un único código base. Cambiar de proyecto cambia la carpeta en la que trabaja el agente, junto con las sesiones y los ajustes de ese proyecto.

## Añade un proyecto

Puedes añadir un proyecto desde varios lugares:

- la entrada **Add project** en la paleta de comandos
- el botón **+** en la parte superior de la barra lateral de sesiones
- el explorador de carpetas cuando eliges un directorio

Apúntalo a una carpeta y OpenChamber la recuerda. El nombre viene de la carpeta; puedes cambiarlo más tarde.

## Cambia de proyecto

Elige un proyecto de la barra lateral para activarlo. Todo —sesiones, git, notas— sigue al proyecto que tengas abierto.

## Haz reconocible un proyecto

Abre **Settings → Projects** para darle a un proyecto un nombre, color o icono personalizado. OpenChamber intenta encontrar un icono automáticamente; consulta [Iconos de proyecto](/es/project-icons/).

> En VS Code, OpenChamber siempre usa la carpeta que tienes abierta como único proyecto, así que no hay nada que añadir ni cambiar. La página de ajustes de proyectos está oculta allí.

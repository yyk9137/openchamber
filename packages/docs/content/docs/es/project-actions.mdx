# Acciones de proyecto

Una acción de proyecto es un comando de shell que guardas una vez y ejecutas con un clic: tu servidor de desarrollo, una compilación, una ejecución de pruebas. Cada proyecto mantiene su propia lista. Configúralas en **Settings → Projects → Project Actions**.

## Añade una acción

1. Abre **Settings → Projects** y busca la sección **Project Actions**.
2. Añade una acción, dale un nombre, elige un icono e introduce el comando a ejecutar.
3. Guarda.

Puedes limitar una acción a ciertos sistemas operativos si un comando solo tiene sentido en uno.

## Ejecuta una acción

Las acciones viven en un menú del encabezado de la app. Haz clic en una y OpenChamber la ejecuta en una terminal dentro de la carpeta de tu proyecto, cambiándote a la vista de terminal para que veas la salida. Deténla desde el mismo menú.

## Abre automáticamente un servidor de desarrollo

Activa **auto-open URL** para una acción que inicia un servidor. OpenChamber observa la salida en busca de una dirección local y se ofrece a abrirla; consulta [Vista previa y servidores de desarrollo](/es/preview/). En el escritorio también puedes enrutarla a través de un reenvío de puertos SSH.
